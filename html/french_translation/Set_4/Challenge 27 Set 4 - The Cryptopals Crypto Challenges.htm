Récupérer la clef d'un CBC où IV=clef

Prenez votre code créé pour l'exercice CBC et modifiez-le pour que la clef utilisée pour le chiffrement CBC soit identique au vecteur d'initialisation (IV).

Nombre d'applications utilisent la clef comme vecteur d'initialisation en imaginant que l'émetteur et destinataire partagent une clef commune, et que cela permet d'économiser de l'espace s'il n'est pas nécessaire de stocker une clef et un IV de manière séparée.

Utiliser la clef comme IV est un risque de sécurité majeur; un attaquant peut modifier le texte chiffré pendant sa transmission et peut pousser le destinataire à déchiffrer une valeur qui lui permettra d'obtenir la clef.

Le code CBC provenant de l'exercice 16 permet de chiffrer une chaîne URL. Vérifiez que chaque octet du texte clair ne contient que des caractères strictement ASCII (ie recherchez toute valeur ASCII ne respectant pas un encodage strict sur 7 bits). Les messages non-conformes doivent provoquer une exception ou renvoyer un code d'erreur qui doit être accompagné du texte clair déchiffré (cela survient très fréquemment au sein de véritables systèmes cryptographiques en usage).

Utilisez votre code pour chiffrer un message d'au moins 3 blocs de longueur : 

AES-CBC(P_1, P_2, P_3) -> C_1, C_2, C_3

Modifiez le message (vous êtes maintenant l'attaquant) : 

C_1, C_2, C_3 -> C_1, 0, C_1

Déchiffrez le message (vous êtes maintenant le destinataire) et lever l'erreur appropriée si un caractère ASCII invalide esst détecté.

En tant qu'attaquant, récupérez le texte clair provenant de l'erreur, et réalisez l'extraction de la clef : 

P'_1 XOR P'_3
