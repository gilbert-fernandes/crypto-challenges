Implémenter une attaque RSA broadcast avec E=3

Supposez que vous êtes un développeur Javascript. C'est à dire que vous utilisez une implémentation naïve de RSA que vous utilisez sans remplissage.

Supposons qu'on vous oblige à chiffrer le même texte clair trois fois de suite, à l'aide de trois clef publiques différentes. Vous le pouvez; c'est déjà arrivé.

Il est alors triviallement possible à un attaquant de déchiffrer votre message, en :

  1. Capturant les trois textes chiffrés et les clefs publiques correspondantes
  2. Utilisez le CRT pour résoudre le nombre réprésenté par les trois textes chiffrés (qui sont des résidus modulo leurs clefs publiques respectives).
  3. Prenez la racine carrée du nombre obtenu

Le CRT dit que vous pouvez prendre tout nombre et le représenter comme la combinaison d'une série de résidus modulo une série de modulos. Dans le cas des trois résidus, vous avez : 

résultat =
  (c_0 * m_s_0 * invmod(m_s_0, n_0)) +
  (c_1 * m_s_1 * invmod(m_s_1, n_1)) +
  (c_2 * m_s_2 * invmod(m_s_2, n_2)) mod N_012

où : 

 c_0, c_1, c_2 sont les trois résidus modulo n_0, n_1, n_2

 m_s_n (pour n dans 0, 1, 2) le produit des modulo
 A L'EXCEPTION de n_n --- ie, m_s_1 est n_0 * n_2

 N_012 est le produit de ces trois modulo

Pour déchiffrer du RSA à l'aide d'une simple racine cubique, sautez la dernière étape finale de modulo; utilisez directement le résultat accumulé et appliquez une racine cubique.
