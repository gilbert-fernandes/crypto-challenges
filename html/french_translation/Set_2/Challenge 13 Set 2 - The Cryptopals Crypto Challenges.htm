Couper-coller ECB

Ecrivez une routine pour parser des structures de la forme k=v Cette routine devra prendre en entrée : 

foo=bar&baz=qux&zap=zazzle

... et produire : 

{
  foo: 'bar',
  baz: 'qux',
  zap: 'zazzle'
}

(récupérer l'objet transmis en paramètre; peu m'importe si vous utilisez du JSON si vous préférez).

Ecrivez ensuite une fonction qui permet de stocker un profil utilisateur dans ce format, quand on lui transmet une adresse email. Quelque chose comme : 

profile_for("foo@bar.com")

... et qui doit produire : 

{
  email: 'foo@bar.com',
  uid: 10,
  role: 'user'
}

... et qui sera codé sous la forme : 

email=foo@bar.com&uid=10&role=user

Votre fonction "profile_for" ne doit pas permettre l'utilisation de méta-caractères (& and =). Faites-le disparaître, placez-le entre guillemets simples, comme bon vous semble mais ne laissez pas les utilisateurs utilisez d'adresse du genre "foo@bar.com&role=admin".

Puis écrivez deux fonctions supplémentaires, faciles : générez une clef AES aléatoire puis :

  1. Chiffrez le profile utilisateur encodé sous la clef; "fournissez" ce résultat à "l'attaquant"
  2. Déchiffrez le profil utilisateur encodé et parsez-le.

Utilisez uniquement les entrées utilisateur pour alimenter profile_for() comme une sorte d'oracle pour générer des textes chiffrés "valides" et les textes chiffrés eux-mêmes. Créez également un profil role=admin
