Altération des paramètres DSA

Prenez votre code DSA de l'exercice précédent. Imaginez-le comme partie d'un algorithme où le client a le droit de proposer des paramètres de domaine (les modulo p et q et le générateur g).

Ce serait un risque majeur, car des attaquants pourraient tromper leurs victimes pour leur faire accepter de mauvais paramètres. Vaudenay a donné deux exemples de mauvais générateurs de paramètres : les générateurs qui sont 0 mod p, et ceux qui sont 1 mod p.

Utilisez les paramètres de l'exercice précédent, mais substituez la valeur 0 pour "g". Générez une signature. Vous allez remarquer quelque chose de très mauvais. Vérifiez la signature. Maintenant, vérifiez toute autre signature, pour tout autre chaîne.

Maintenant essayez (p+1) comme valeur de "g". Avec ce "g" vous pouvez générer une signature magique s, r pour toute clef DSA publique qui sera capable de valider vis à vis de toute chaîne. Pour un z choisi arbitrairement :
